Что такое git workflow

Workflow это план работы по которому программисты работают вместе над проектом при этом не мешая друг другу

Для чего он нужен ?

Чтобы каждый программист мог спокойно делать свою часть работы и при этом чтобы изменения в проекте не мешали тому что уже работает

Как он работает?

Получается есть основная ветка например master, в которой хранится самая важная и уже готовая версия проекта. Для новой задачи создается своя отдельная ветка например feature, где работают над своей частью, чтобы не трогать основную. Когда задача готова эту ветку проверяют и объединяют с основной веткой.

Что такое git rebase

Отличие между git rebase и git merge

Git merge просто обьединяет две ветки создавая отдельный коммит , а git rebase не создает коммита слияния а просто перемещает коммиты в основную ветку

Что такое git fetch

Git fetch это команда которая загружает из удаленного репозитория содержимое но не обновляет локальный репозиторий

Разница между git fetch от git pull

Git pull загружает изменения и сразу же обьединяет с текущей веткой а git fetch не применяет их в текущей ветке.
